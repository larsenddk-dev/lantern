Goal:
  Turn the Documents placeholder into a real module: upload files, store them locally, list them, view extracted text, and delete them — with text extraction for .txt/.md (stdlib), .pdf (pypdf) and .docx (python-docx). Backed by SQLite + local file storage on FastAPI, with a working UI on Next.js, following existing conventions. This completes the last v0.1 light module (Memory stays a stub for now).

Scope in:
  - Backend (apps/api): add a `documents` SQLite table in init_db() (id, filename, content_type, size_bytes, extracted_text, created_at) plus db_* helpers, following existing conventions.
  - Add python-multipart, pypdf, python-docx to requirements.txt and install them into the api venv.
  - Upload endpoint: POST /documents accepting a multipart file; sanitize the filename (no path traversal), enforce a sane max size, store bytes under data/uploads/<id>/, extract text by type (.txt/.md via stdlib decode, .pdf via pypdf, .docx via python-docx; unsupported types stored with empty extracted_text), and persist a row.
  - Read/list/delete endpoints: GET /documents (metadata list, no full text), GET /documents/{id} (metadata + extracted_text), DELETE /documents/{id} (remove the row AND the stored file).
  - Frontend types + client: add Document types/payloads to lib/types.ts and methods to lib/api.ts (upload via FormData, list, get, delete).
  - Documents page (app/documents/page.tsx): replace the stub with a UI to upload a file, see the list (name, type, size, date), open a document to view its extracted text, and delete it.
  - Tests (apps/api/test_main.py): add offline pytest for upload + extraction (at least .txt and .md asserting extracted text; .docx generated in-test via python-docx asserting extraction; .pdf upload/list/get/delete works with extraction running without error) and list/get/delete + filename sanitization; keep all prior tests passing, no network/secrets.
  - Docs/state: update README (Documents now live, new deps noted) and .nogra/state/SESSION-CHECKPOINT.md + CURRENT-TASKS.md (Phase 2c done; name next phase = Memory).

Scope out:
  - OCR for scanned/image PDFs.
  - Embeddings, vector search, RAG, or chatting with a document's content (later Memory/RAG phase).
  - The Memory module (stays a stub).
  - Any change to chat/provider/sessions/notes/tasks behavior beyond additive, non-breaking edits.
  - New dependencies beyond python-multipart, pypdf, python-docx.
  - Auth/multi-user, deployment, cloud storage — files stay local.
