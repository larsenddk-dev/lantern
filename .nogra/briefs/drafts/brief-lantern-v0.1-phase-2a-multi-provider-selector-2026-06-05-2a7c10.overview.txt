Title: Lantern v0.1 — Phase 2a: multi-provider selector (choose provider + model at runtime)
Brief id: brief-lantern-v0.1-phase-2a-multi-provider-selector-2026-06-05-2a7c10
Status: ready
Workspace: lantern
Target model: anthropic:sonnet

Goal:
  Let the user choose which OpenAI-compatible provider and model the chat uses at runtime — managed in Settings, persisted, and applied to every chat send — so Lantern can talk to any free provider (OpenRouter, Gemini, Groq, Mistral, Cerebras) or local Ollama without editing code. This is Phase 2a of v0.1; the real Notes/Tasks/Documents modules are a later brief (Phase 2b).

Scope in:
  - Backend: persist named provider configs in SQLite (label, baseUrl, model, apiKey) with an 'active' selection; GET/PUT/DELETE endpoints; store under the gitignored local data dir.
  - Backend: /chat resolves the active provider/model (or accepts providerId/model in the request body) instead of env-only; environment stays as the default fallback.
  - Backend: never return stored API keys in full — mask them on read; never write keys to a git-tracked file.
  - Web: Settings page UI to add/edit/delete provider configs and choose the active provider + model, persisted via the API.
  - Web: a provider/model indicator and quick switcher in the chat header that reads the active selection; chat send uses the active provider/model.
  - Web: client API helpers in apps/web/lib for the new settings/providers endpoints, with matching TypeScript types.
  - Tests: pytest for providers CRUD + active-selection persistence round-trip, /chat streaming using the selected provider/model against a STUB provider (no network/secrets), and API-key masking on read.
  - Docs/state: update .env.example and README provider-setup notes; update .nogra/state/SESSION-CHECKPOINT.md and CURRENT-TASKS.md (Phase 2a done; name Phase 2b = Notes/Tasks/Documents, Phase 3 = Memory/RAG).

Scope out:
  - Real Notes/Tasks/Documents module features (Phase 2b).
  - Memory/RAG retrieval (Phase 3), Agent + tools, Deep Research, Compare, Email, Calendar, Cookbook/model-serving, image editor, PWA.
  - Auth/multi-user, deployment/hosting/Docker.
  - Calling live external AI providers in automated tests, or adding real API keys/secrets to the repo.
